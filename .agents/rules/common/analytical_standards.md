# Analytical Standards

## 1. Code Review Principles

- Prioritize correctness and security.
- Ensure all logic is covered by unit tests.
- Maintain high information density in logs.

## 2. Logical Evaluation
- Always check for edge cases.
- Use guard clauses to reduce nesting.
- Avoid side effects in pure functions.

## 3. Structural Integrity
- Keep files focused on a single responsibility.
- Use consistent naming conventions across the project.
- Document complex logic with clear comments.

## 4. Documentation
- Keep documentation up to date with code changes.
- Use clear and concise language.
- Provide examples for complex APIs.
