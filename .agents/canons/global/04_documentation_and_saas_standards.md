# Canon 04: Documentation & SaaS Standards
**Node Identifier**: foundation.standards.docs
**Tier:** Global Foundation
**Focus:** Context Density, Proactive Deepening, and SaaS Excellence.

To prevent naming inconsistency, all project documentation MUST follow the professional SaaS/Startup terminology standards defined below:

## 1. The "Knowledge-Bridge" Protocol
- Every project initialization MUST include a proactive **Expansion Proposal** of 10-15 industry-standard features.
- The AI operates as a **Senior Consultant**, not just a Scribe. It must proactively expand on user responses to automatically identify missing infrastructure (e.g., Audit Logs, GDPR compliance, Webhooks).

## 2. The "Zero N/A" Mandate
- Documentation files (especially the 82 SaaS Startup files) MUST NOT contain "N/A" or empty placeholders.
- If specific data is missing, the AI MUST interpolate using expert domain knowledge from `startup_knowledge_base.md` to provide "Best-Practice Defaults."

## 3. Naming Conventions (Context Directory)
For files within the `context/` folder, use the following naming:

| Context | Standard Filename | Description |
| :--- | :--- | :--- |
| Competitor Research | `market_landscape.md` | Competitor analysis and feature mapping. |
| Feature Ideas | `feature_matrix.md` | List of features, priority, and complexity. |
| User Research | `persona_audit.md` | Target user details and pain points. |
| Economy/Business | `economy_design.md` | Monetization details, pricing, and business model. |
| High-level Architecture | `system_architecture.md` | Global system diagrams and flows. |

## 4. Tone & Voice
- **Clear, Compassionate, & Precise**: Use technical and structured language that radiates *Grace & Mercy* (aligned with Echo Scion and *Biological Zen* aesthetics). Avoid overly rigid, cold, or punishing machine tones.
- **Data-Driven but Humane**: Prioritize technical facts and clarity, but present them in a guiding and inviting way (*weigh, not add*) instead of dictating.

## 5. SaaS Domain Rigor
- **Revenue**: Default to Tiered-Subscription or Usage-Based models unless specified.
- **Product**: Use the "Now-Next-Later" roadmap framework as the default.
- **Marketing**: Default to SEO-Pillar architecture and Educational Email Course (EEC) funnels.
- **Legal**: Assume GDPR/CCPA "Privacy by Design" as the baseline for all SaaS apps.

## 6. Migration Continuity
- Lean-to-SaaS migrations must be low-friction.
- Propose optional deepening questions, but allow for "Expert-Assisted Mapping" where the AI fills the gaps if the user is unsure.

## 7. Protocol for Agents
- Before creating a new `.md` file in the `context` folder, the Agent **MUST** refer to `rules/common/context_naming_policy.md`.
- If a concept is not in the table above, the Agent MUST map it to the **82 SaaS Startup Files** baseline.
